# Транспортная задача в проекте Farm Sales

## 1. Что решает алгоритм

Алгоритм нужен для роли логиста. Его задача:

- взять все заказы со статусом `APPROVED`;
- распределить их между тремя водителями;
- учесть ограничения машины по весу и объёму;
- построить порядок посещения точек доставки;
- вернуть логисту готовый маршрутный план для подтверждения.

## 2. Где реализовано

Основная логика находится в backend:

- `POST /api/orders/auto-assign/preview` — построить план без записи в БД
- `POST /api/orders/auto-assign/approve` — применить подтверждённый план
- `POST /api/orders/auto-assign` — сразу построить и применить план

Ключевой класс: `backend/src/main/java/com/farm/sales/service/OrderService.java`

## 3. Какие данные подаются на вход

Алгоритм использует:

- заказы в статусе `APPROVED`;
- координаты доставки каждого заказа;
- список водителей с ролью `DRIVER`;
- текущую активную загрузку водителей по заказам `ASSIGNED`;
- вес и объём товаров из состава каждого заказа.

## 4. Основная идея решения

В проекте используется не точный математический solver, а эвристический алгоритм.

Он состоит из двух частей:

1. распределение точек доставки по водителям;
2. построение маршрута внутри каждого водителя.

## 5. Как распределяются заказы по водителям

### Шаг 1. Отбор водителей

Система берёт только первых трёх водителей. Это жёсткое ограничение демонстрационной версии.

### Шаг 2. Учёт текущей нагрузки

Для каждого водителя считается, сколько у него уже есть активных заказов в статусе `ASSIGNED`.

После этого система старается выдать больше новых заказов тем водителям, которые сейчас менее загружены.

### Шаг 3. Группировка заказов по координатам

Если несколько заказов имеют одинаковые координаты доставки, они объединяются в одну логистическую точку.

Для этой точки суммируются:

- общий вес;
- общий объём;
- список заказов в точке.

### Шаг 4. Жадная загрузка машин

Все точки сортируются по расстоянию до общего логистического хаба в Могилёве.

Дальше алгоритм по очереди пытается загрузить точки:

- сначала в первую машину;
- затем во вторую;
- затем в третью.

Точка назначается водителю, только если одновременно выполняются условия:

- не превышен лимит по числу заказов для этого водителя;
- не превышен лимит по весу;
- не превышен лимит по объёму.

Если точка не помещается, она остаётся нераспределённой.

## 6. Как строится маршрут

Когда точки уже распределены по водителям, для каждой машины строится порядок объезда.

Используется метод ближайшего соседа:

- есть текущая точка;
- среди оставшихся выбирается ближайшая;
- она становится следующей остановкой;
- затем алгоритм повторяется для новой текущей точки.

Если в одной точке лежит несколько заказов, они ставятся подряд.

## 7. Как считается расстояние

Расстояние между точками считается по дорогам через OSRM.

Это значит:

- ближайшая точка выбирается не по прямой, а по реальному дорожному графу;
- порядок остановок стал ближе к настоящему маршруту водителя;
- если роутинг-сервис временно недоступен, система может использовать fallback-расчёт по координатам, чтобы автоназначение не ломалось полностью.

## 8. Ограничения модели

В текущей версии используются фиксированные ограничения:

- максимум 3 водителя;
- максимум `1500 кг` на машину;
- максимум `12 м3` на машину;
- общий логистический хаб находится в Могилёве;
- в демонстрационной версии используется внешний публичный OSRM-сервис, поэтому качество маршрута зависит и от его доступности;
- возврат машины обратно на склад не учитывается.

## 9. Что получает логист

В preview логист видит:

- сколько всего одобренных заказов;
- сколько удалось распределить;
- сколько осталось нераспределёнными;
- суммарную оценку длины маршрутов;
- отдельный маршрут по каждому водителю;
- порядок остановок внутри каждого маршрута.

После подтверждения плана:

- в заказ записывается назначенный водитель;
- статус меняется на `ASSIGNED`;
- создаются события в timeline;
- пишется аудит;
- отправляются уведомления другим ролям.

## 10. Как коротко объяснять преподавателю

Транспортная задача в проекте решается эвристически.

Система берёт все одобренные заказы, учитывает текущую загрузку трёх водителей, ограничения машины по весу и объёму, жадно распределяет точки доставки между машинами, а затем внутри каждого маршрута строит порядок объезда по методу ближайшего соседа.

Этого достаточно для учебной системы и демонстрации логистической автоматизации в курсовом проекте.
