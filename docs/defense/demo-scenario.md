# Demo Scenario For Defense

## 1. Цель

Этот сценарий нужен для показа проекта преподавателю в предсказуемом состоянии, независимо от того, какие действия уже выполнялись в системе ранее.

## 2. Как запустить reset

### Локально

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\demo-reset.ps1 -Base http://127.0.0.1:8080/api
```

### На публичном backend

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\demo-reset.ps1 -Base https://farm-sales-backend.onrender.com/api
```

## 3. Что делает reset

- полностью очищает накопленные заказы и позиции заказов;
- удаляет timeline, аудит, складские движения и realtime-уведомления;
- удаляет изменённые demo-данные и пересоздаёт канонических demo-пользователей;
- заново поднимает каталог и demo-пользователей;
- возвращает чистую стартовую точку: `0` заказов и `0` точек доставки.

## 5. Рекомендуемый порядок показа

### Шаг 1. Директор

- войти под `berezka`
- показать профиль
- добавить первую точку доставки
- открыть каталог
- добавить товары в корзину
- отправить новую заявку

### Шаг 2. Менеджер

- войти под `manager`
- открыть `Сводка`
- показать аналитику и категории спроса
- открыть `Заявки`
- показать карточку неодобренного заказа: адрес, сумму, количество позиций и состав
- одобрить новую заявку директора
- открыть `Отчёты` и выгрузить Excel
- при необходимости показать создание нового директора

### Шаг 3. Логист

- войти под `logistician`
- показать список `APPROVED` заказов
- открыть автоназначение
- применить маршрутный план

### Шаг 4. Водитель

- войти под `driver1`
- открыть маршрут
- показать назначенный заказ
- отметить его доставленным

## 6. Что важно проговорить на защите

- система поддерживает четыре роли с разными зонами ответственности;
- заказ проходит через полный жизненный цикл;
- backend ведёт аудит и timeline;
- frontend получает realtime-обновления;
- проект развёрнут публично и содержит механизм reset демонстрационных данных.
